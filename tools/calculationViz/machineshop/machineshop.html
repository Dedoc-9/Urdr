The kit

A dedicated machine-shop authoring surface: build a wireframe object (grid snap, exact coordinate editor, mirror for symmetry, live edge-length + bounding-box measurements, multiple objects into one file), then bridge it into the game frontend — the Urðr Designer — through the verified admission bridge. It reuses one engine (no duplication), so nothing here re-implements the CAD or the bridge; it just focuses them for shop work.

Author → game, in four steps

  1. Build/measure the object in the editor (grid snap + the coordinate editor for exact, symmetric layouts).
  2. Save JSON → complex.urdr.json (a URDR-COMPLEX-1). Or use the in-editor “Export → game world (grounded)” button, which does the bridge step in-browser (byte-identical geometry to the script).
  3. Bridge it to a game object, grounded to the floor: 
    python3 tools/calculationViz/bridge_to_world.py complex.urdr.json world.json --rest-face
    Grounding: default is lowest-point; --rest-face sits it flat on a stable face. Coordinates are integer-snapped at this authoring boundary — a float in the export is WORLD-REFUSE; the runtime never rounds.
  4. Open it in the Designer / gated netcode: 
    python3 tools/editor/load_world.py world.json frame.pgm     # Designer renders it (URDRFB1)
# or feed a --body dynamic world to worldstep.world_from_export (gated N4)

Honest boundary

The kit is presentation (browser float, off-gate). The bridge carries geometry + placement only — the object's shape and where it stands. It does not carry topology: Betti / witness do not travel into the game world; that stays behind verify.py. “Welded to the frontend” means joined by the verified bridge, not fused code — one engine, one source of truth.

grade: PREVIEW / NOT_MEASURED · off-gate · the game authority is verify.py + worldstep
