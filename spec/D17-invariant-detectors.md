# D17 ‚Äî The invariant-detector admission law (what survives?)

Status: **META-CONTRACT.** D17 does not define a subsystem. It names the *admission law* that
every gated detector in this project has already been following, and makes that law explicit and
falsifiable. Its correctness test is **retro-admission**: the existing detectors (D14, D15, D16,
rigidity, criticality) must each satisfy it ‚Äî *if even one cannot be expressed under D17, D17 is
wrong.* That test was run against the committed gate (`e390f42`) and all five pass; the mapping is
¬ß6. This is an unusually strong specification because it is falsifiable by code that already exists.

The organizing question is not a prover's ("is proposition P true?") nor a CAS's ("compute f(x)?")
but:

> **Which exact statements about an object remain true under a declared class of transformations?**

An invariant is a value that is *constant on the orbits of that class*. A detector computes one, on
a domain where it can, and **refuses** everywhere else.

## 1. The detector

A detector is a tuple

    ùíü = (Dom, Inv, W, ~, R)

- **Dom** ‚Äî a **decidable** domain. Membership is checkable.
- **Inv : Dom ‚Üí V** ‚Äî an **exact** invariant map (no rounding, or bounded-regime honesty).
- **W** ‚Äî an **independently checkable witness** that recomputes / certifies the value.
- **~** ‚Äî a **declared equivalence relation** on Dom, generated by a declared class of *admissible
  transformations* (morphisms). It need NOT be a group action: a group (invertible) is the special
  case; monoids (non-invertible evolution, presentation edits) and bare equivalences (provenance,
  partition choices) are the general case. `Inv` is invariant iff it **factors through the quotient
  `Dom/~`** ‚Äî i.e. `Inv(x) = Inv(y)` whenever `x ~ y`.
- **R** ‚Äî a **total refusal relation** outside Dom: every object not in Dom gets a *typed* refusal,
  never a wrong answer.

## 2. The admission law

A detector is **admitted** iff:

1. **Membership is decidable** ‚Äî `x ‚àà Dom?` is computable.
2. **REFUSE is total outside the domain** ‚Äî `x ‚àâ Dom ‚áí R` fires, typed.
3. **The invariant is exactly computable** ‚Äî `Inv` is deterministic and exact.
4. **The witness recomputes the invariant** ‚Äî (4a) self-verifies the value; and, for the *cross-placed*
   grade, (4b) an independent placement reproduces it (the D8 move).
5. **The invariant is constant on a declared corpus of `~`-transformations** ‚Äî `Inv(T¬∑x) = Inv(x)`.
6. **A deliberately non-invariant quantity reddens the gate** ‚Äî a `Q` with `~ ‚äÑ ker(Q)` must fail,
   proving the invariance is load-bearing and not vacuous.

Nothing more, nothing less ‚Äî and nowhere does it mention topology, physics, graphics, or
networking. That silence is why it generalizes.

**Every theorem ships with its counterexample.** Conditions 2 and 6 are first-class, not
afterthoughts: admission is measured as a **positive corpus + a negative corpus + typed refusals**,
which is precisely what a well-formed gate stage in this repo already records ‚Äî
`golden` ¬∑ `invariance property` ¬∑ `-selftest` (non-invariance) ¬∑ `-refusal`.

## 3. Two relations, and completeness as a graded claim

A detector carries two relations: the **declared** `~` (what it must identify) and the **induced
kernel** `ker(Inv) = {(x,y) : Inv(x) = Inv(y)}` (what it actually identifies). Admission requires
`~ ‚äÜ ker(Inv)`.

The detector is **complete** iff `ker(Inv) = ~` ‚Äî the invariant separates *exactly* the orbits, so
`Inv(x) = Inv(y) ‚ü∫ x ~ y`. Completeness is a **separate** claim from admission and is frequently
undecidable (deciding orbit-equality can be harder than computing the invariant). It is therefore
**graded** like everything else ‚Äî `PARTIAL`, `MEASURED-complete` (on a corpus), or `PROVEN-complete`
‚Äî and never a free label. Complete detectors are terminal for their `~`: every weaker detector on
the same domain **factors through** them.

## 4. The lattice of detectors

Order detectors by their induced kernel: a finer kernel distinguishes more. If `~‚ÇÅ ‚äÜ ~‚ÇÇ` then
`Inv‚ÇÇ` is *coarser* (`Inv‚ÇÇ(x)=Inv‚ÇÇ(y)` is a weaker statement ‚Äî more is identified). This is a genuine
partial order: some detectors distinguish more, some deliberately forget more, and complete
invariants sit at the bottom of their domain's order. Organizing the library *is* building this
lattice.

## 5. The invariant profile (the API)

An object does not have *an* invariant; it has a **profile** ‚Äî the table

    { detector ‚Üí value | REFUSE }

Each detector answers where it is competent and refuses where it is not. Example (illustrative):

    URDROBJ2 object
      canonical (D14):   digest 3c36067‚Ä¶
      rigidity:          RIGID (dof 0)
      topology (H‚ÇÅ):     REFUSE  (not a chain complex)
      criticality:       REFUSE  (not a population field)
      symmetry:          REFUSE  (no detector admitted yet)

Nothing speculative, nothing inferred ‚Äî exact detectors answering only where competent. This is the
honest form of "map the invariant landscape of an object": **not** universal introspection (which is
undecidable ‚Äî no algorithm returns the symmetry group or conserved quantities of an *arbitrary*
object), but a **finite library of detectors with decidable domains and total REFUSE**, whose union
of non-refusing answers is the profile. REFUSE does the same work here that `FIELD-REFUSE` and
`WORLD-REFUSE` do everywhere else: it lets the engine say *"I can't tell, from here,"* which is what
keeps *"what survives?"* honest.

## 7. Self-application

**"Evidence Against C8" (D5) is D17 applied to the engine's own primitive set.** The object is the
sealed alphabet; the admissible transformation is *"admit a new domain"*; the invariant under test
is *"the primitive set is unchanged"*; the ledger is the corpus of orbits checked. The engine
already runs its central question on itself ‚Äî and every clean composition is one more orbit on which
the alphabet proved invariant.

## 8. Admission ladder

1. **State the law** (this document) and **retro-admit** the existing detectors (¬ß6 ‚Äî done, and the
   validation *is* the proof the law is right).
2. **A structural lint** (`invariant_detectors` gate stage) ‚Äî **LANDED.** Each admitted detector
   *declares* (in an explicit manifest, never inferred from row-name conventions) which recorded rows
   fill its four roles; the lint checks every role is present, every named row was recorded, and every
   such row passed, with a non-vacuity self-test that rejects a broken declaration. Building it fixed a
   real gap ‚Äî rigidity gained a `rigidity-verdict-refusal` row (overflow ‚Üí `REFUSE`) ‚Äî so all five
   detectors are mechanically D17-compliant. The admission law is now a gate row, not just a table.

   `tools/intla/toric.py` (+ the new ùîΩ‚ÇÇ substrate `gf2.py`): `Dom` = a cellulated surface as ùîΩ‚ÇÇ
   boundary maps with `‚àÇ‚ÇÅ‚àÇ‚ÇÇ = 0` (a non-complex ‚Üí `TORIC-REFUSE`); `Inv` = code dimension
   `k = dim H‚ÇÅ = |E| ‚àí rank ‚àÇ‚ÇÅ ‚àí rank ‚àÇ‚ÇÇ` (`= 2¬∑genus`); `W` = boundary matrices ‚Üí ranks ‚Üí the H‚ÇÅ
   dimension, with digest `391e49e5‚Ä¶`; `~` = cellulation refinement / stabilizer-preserving ops (k is
   genus-invariant). It was admitted by the *same* six conditions ‚Äî a new exact substrate (ùîΩ‚ÇÇ), an
   algebraic invariant, and a multi-part witness, with **no flex to the contract**. The
